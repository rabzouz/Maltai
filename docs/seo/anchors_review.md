# Revue des ancres internes — homepage

## Constat (avant)
Ancres répétées détectées :
- `Connexion` ×3 (header, CTA final, footer)
- `Ouvrir l'app` ×2
- `GitHub` ×2

## Analyse
Ces répétitions sont **fonctionnelles** (navigation, conversion) et n'ont rien
d'abusif. La priorité SEO est de **ne pas sur-optimiser** : on garde une UX claire.

## Actions retenues
- **Conserver** `Connexion` et `Ouvrir l'app` : libellés d'action attendus par
  l'utilisateur, la diversité artificielle nuirait à l'UX.
- **Diversifier les liens sortants GitHub** quand le contexte le permet :
  - hero : `GitHub` → conserver (action courte)
  - section Installer : `Voir le dépôt GitHub` ✔ (déjà différencié)
  - footer : `GitHub` → peut devenir `Code source (GitHub)`.
- Les liens de release sont déjà variés (`Télécharger le ZIP`, `Voir la release`,
  `Télécharger Maltai`, `Voir le dépôt GitHub`).

## Recommandation
Aucune sur-optimisation. Une seule micro-amélioration possible (footer) ; impact
SEO marginal, à faire seulement si cela n'alourdit pas l'UX.
